- Εντολή μεταγλώττισης και εκτέλεσης προγράμματος: 
    -> ./run

- Παραδοχές / Διευκρινήσεις:
   -> Δεν έχουμε βάλει στις συναρτήσεις του DeckBuilder το είδος της κάρτας σαν όρισμα στους constructors των card, 
       αλλά το αρχικοποιούμε στους constructors των καρτών μέσω της setType() και των enum.
    
   -> Όταν το πρόγραμμα δέχεται είσοδο αριθμό από τον χρήστη και πατηθεί άλλος χαρακτήρας, 
       τότε ενδέχεται να τερματίσει ανώμαλα

+ Starting Phase:
   -> Αρχικοποιούνται οι 2 παίκτες. Συγκεκριμένα, τα Deck, Ηand, Provinces, Army και Holdings.

   -> Εκτός από τα ζητούμενα, στην αρχή του γύρου δίνεται στον παίκτη gold,
      αναλόγως με το συνολικό Harvest Value όλων των Holdings του.

   -> Για κάθε παίκτη, ορίζουμε ένα instance deckBuilder και ορίζουμε τα deck του αναλόγως.

+ Εquip Phase:
   -> Ο παίκτης επιλέγει Personality από το army του και κανει attach Followers και Items από το Hand του.

+ Battle Phase:
   -> Ο κάθε παίκτης, στο prepareBattle, ορίζει ποια Personalities θα γίνουν tapped, 
      τα οποία και θα αμυνθούν και θα επιτεθούν ανάλογα με τη σειρά παιξίματος μέσα στην battle.

   -> Ο νικητής κάθε φορά επιλέγεται ανάλογα με τις περιπτώσεις που δίνονται στην εκφώνηση.

+ Economy Phase:
   -> Ο κάθε παίκτης επιλέγει να αγοράσει από τα Provinces του Personalities ή Holdings και να τα προσθέτει    
       στο army ή holdings του, αναλόγως με τον τύπο της κάρτας. Έπειτα προστίθεται ένα unrevealed Province     
       στη θέση του.

  -> Στην περίπτωση που η κάρτα είναι τύπου Mine, GoldMine, CrystalMine, καλούνται αυτόματα οι 
       toSubHolding/ toUpperHolding, οι οποίες ελέγχουν αν υπάρχει διαθέσιμο Holding τύπου mine
       που μπορεί να συνδεθεί σαν Sub ή Upper Holding.

  -> Τα Mine, CrystalMine μπορουν να συνδεθούν μεταξύ τους, αν υπάρχει ήδη σύνδεση με GoldMine.
       Αναλόγως με τη σύνδεση, αυξάνεται και το harvest value του παίκτη όπως διατυπώνεται στην εκφώνηση.

+ Final Phase:
   -> Εκτυπώνεται η κατάσταση του Παιχνιδιού.
